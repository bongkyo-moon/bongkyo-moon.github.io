Operating Systems

Text: Gary Nutt, "Operating Systems", 3rd Edition, Addition Wesley, 2004

Objectives - To answer and develop the ability to answer the following questions: What is the purpose of an operating system? How are operating systems constructed? Introduction: resource manager view, virtual machine view, security; types of operating system. Machine organisation and input/output: polled input/output, interrupt driven input/output, device driver structure. The system nucleus (kernel): context switching, first level interrupt handling, kernel implementation of processes, kernel implementation of semaphores. Concurrent processes: interleaving, non-determinism, process interaction - sharing, synchronisation, communication, locks, semaphores, monitors. Scheduling: priority - pre-emption, run to completion, time sliced, multi-level queues. Program construction utilities: assembler, link editor, relocation. Memory management: single contiguous store allocation and overlays, fixed partition store allocation, dynamic partition store allocation and fragmentation/compaction, virtual addressing, memory management policies. Virtual memory: address translation, page faults, replacement policies, working sets, swapping.

Advanced System Programming

Text1: Mark Mitchell, Jeffrey Oldham, and Alex Samuel, "Advanced Linux Programming", 1st Edition, New Riders, 2001
Text2: N. Metthew and R. Stones, "Beginning Linux Programming", 3rd Edition, Wiley, 2004

Objectives - To help already experienced developer with programming for the GNU/Linux system, and to learn about some of its advanced features and capabilities. To write more sophisticated programs with features such as multiprocessing, multithreading, interprocess communication, and interaction with hardware devices. To improve your programs by making them run faster, more reliably, and more securely, or by designing them to interact better with the rest of the GNU/Linux system.

Assembly Language and System Programming

Text1: Leland L. Beck, "System Software - An Introduction to System Programming", 3rd Edition, Addison-Wesley, 1997
Text2: Kip R. Irvine, "Assembly Language for Intel-based Computers", 4th Edition, Prentice Hall, 2003

Objectives - It is important to teach students how to write and debug programs at the machine level. It does give students the first-hand experience of writing software in an environment that teaches them how the computer really works. Students will have a memorable experience that gives them the confidence to work in any OS/machine-oriented environment. The course focuses on the introduction to the design and implementation of various types of system software. The theme of the course is to understand the relationship between machine architecture and system software. System software consists of software programs that support the operation of a computer. A variety of system software programs (listed in the course description) are to be covered in the course. Some other system software programs include data base management systems, and embedded software systems. Some software engineering concepts and issues will also be introduced. A course project provides practical experience to the students in designing and implementing system software.

IP-Based Next-Generation 4G Networks

Text1: Jyh-Cheng Chen and Tao Zhang, "IP-Based Next-Generation Wireless Networks - Systems, Architectures, and Protocols", John Wiley & Sons, 2004
Text2: Dave Wisely, Philip Eardley and Louise Burness,"IP for 3G: Networking Technologies for Mobile Communications", John Wiley & Sons, 2002

Objectives - This course is a practical class of up-to-date information on the growing body of specifications associated with wireless IP network architectures, systems, and protocols, with a focus on the network layer and above. In a clear, understandable manner, the lecturer provides a detailed description and comparison of next-generation wireless networks, specifically IP-based wireless networks, and discuss critical issues that may arise in their design. In addition to network architecture, the course covers protocols in four major areas: Signaling, Mobility, Quality of Service (QoS), and Security.
